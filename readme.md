# AI 编码能力评测 - 记账 App

用标准化的方式评测不同 AI 模型（Claude Opus、GLM、Minimax 等）的编码能力。

**评测方法**：

- 通过在 Claude Code 中切换不同模型进行对比
- 让 AI 设计一款个人记账 App，同时实现后端 API + Web 前端 + iOS 移动端
- 只安装基础 skills: superpowers、frontend-design
- 每跑完一个单元 clear 一次上下文，每个模型的测试都放在独立分支，降低相互间干扰

**核心原则**：

- 本地快速验证，不依赖远程服务
- 方案框架适度粗粒度，**留空间给模型自行做技术选型**（选型本身也是评测维度之一）
- 递进任务，从简单到复杂
- 需考察 AI 整体理解、输出能力（包括图片理解、输出）和处理任务耗时情况

⚠️ 注意：虽然尽可能的尝试标准化评测，但是因为各个模型本身特点不同，和工具（Claude Code 及各种 skill）的适配性也存在差异，每次运行得到的结果也存在随机性，**所以评测结果仅供参考**。

## 功能分级

### L1 - 基础骨架

创建一个个人记账 App，包含 Web 前端（Vue 3 + TS）、后端 API（Node.js + TS + SQLite）和 iOS 客户端（Swift + UIKit）。

### L2 - 分类与统计

在现有项目基础上增加以下功能：

1. **分类管理**：预置常用分类（餐饮、交通、购物、工资等），每个分类有 icon 和名称
2. **统计功能**：按月查看收入/支出/结余总额，按分类查看支出占比（需要图表展示）
3. **列表增强**：分页加载 + 按类型/分类/日期范围筛选

### L3 - 多账本与分享

用户可以创建多个账本。账本的 owner 可以通过分享码将账本分享给他人，他人以只读方式加入。用户可以在合并视图中勾选多个账本（自己的 + 已加入的），查看汇总收支统计，从而实现以家庭、团队为单位的财务概览。

### L4 - iOS 离线记账（额外加分测试）

针对 iOS 平台设计离线记账功能，使设备无网络时仍可正常记账（创建/编辑/删除记录）。连接网络后将离线数据同步到服务端，需要考虑同一条记录在离线期间本地和服务端都被修改的冲突处理方案。

## 总体框架

### 10 个单元

```
L1: Backend-L1 -> Web-L1 -> iOS-L1         (3 个单元)
L2: Backend-L2 -> Web-L2 -> iOS-L2         (3 个单元)
L3: Backend-L3 -> Web-L3 -> iOS-L3         (3 个单元)
L4: Backend+iOS-L4                         (1 个单元)
```

### 固定约束（保证可比性）

| 项目     | 固定值                                |
|----------|---------------------------------------|
| 前端框架 | Vue 3 + TypeScript                    |
| 后端语言 | Node.js + TypeScript                  |
| 数据库   | SQLite（零安装）                      |
| iOS      | Swift + UIKit                         |
| 鉴权     | JWT                                   |
| 项目结构 | monorepo: `backend/` `frontend/` `ios/` |

### 留给模型自选（评测选型能力）

后端框架、ORM、前端 UI 库、请求校验库、iOS 布局方式、iOS 网络层、图表库、CSS 方案等。

### 评分体系（每单元 10 分，总计 100 分）

| 维度           | 分值 | 评判标准                                                     |
|----------------|------|--------------------------------------------------------------|
| 可运行性       | 0-2  | 0 = 编译失败，1 = 编译通过但运行有错，2 = 可正常运行          |
| 功能完整       | 0-4  | 按 checklist 通过比例线性计分                                 |
| 代码质量+选型  | 0-2  | 0 = 混乱/选型不合理，1 = 可接受，2 = 分层清晰+选型合理有理由  |
| 安全性         | 0-1  | 有明显安全漏洞 = 0，无 = 1                                    |
| 文档           | 0-1  | 无文档或无用 = 0，能帮助下一单元衔接 = 1                      |

### 额外记录项（不计入单元评分）

| 记录项             | 说明                                                         |
|--------------------|--------------------------------------------------------------|
| 图片理解能力       | 模型对输入图片（截图、设计稿等）的理解和应用情况              |
| 图片生成能力       | 模型生成图片（图标、示意图等）的质量和准确度                  |
| 总耗时与 Token 消耗 | 评测全程总耗时和总 Token 用量                                |
| 人工参与度         | 过程中需要人工介入纠正、补充的频率和程度                      |
